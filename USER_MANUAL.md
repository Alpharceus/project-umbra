# Project Umbra — User Manual & Operation Guide

**Project Umbra** turns spare screens — second monitors, laptops, tablets, TVs —
into a calm, synchronized field of dark, slow, ambient visuals. It's designed to
sit in the background of a movie night or a room: low brightness, low contrast,
slow motion, no flashing, and no need for matching screen sizes or orientations.

It behaves like a **screensaver served over your network**: one PC runs the
server, every other screen is just a web browser pointed at a URL. Nothing is
installed on the other devices.

---

## 1. How it works (read this first)

There are three roles:

1. **Conductor** — the Python server (`umbra.exe` or `python run_umbra.py`). It
   runs on **one** PC, owns all state (theme, brightness, timing, sleep), serves
   the apps over HTTP, and syncs everything over WebSockets.
2. **Players** — the screens. **Any browser** that opens the join URL becomes a
   player and renders the visuals locally. No install, no files copied.
3. **Dashboard** — the remote control, a browser page (`/dashboard`), ideally on
   your phone.

```
              +-------------------------------+
              |  Conductor (only PC with code)|
              |  umbra.exe  ·  port 8000       |
              +-------------------------------+
                            | LAN (HTTP + WebSocket)
        +-------------------+--------------------+
        v                   v                    v
   Player screen       Player screen         Dashboard
   (browser /join)     (browser /join)       (phone /dashboard)
```

**You do NOT copy the project to other devices.** The other screens only need a
browser and the host's URL. The visuals are generated by math in each browser
(procedural), so almost no network bandwidth and **zero stored video**.

## 2. Quick start

### Option A — the packaged app (recommended)
1. Double-click **`umbra.exe`** (inside the `dist/umbra/` folder, which also
   contains `_internal/` and `data/` — keep them together).
2. The **control dashboard opens automatically** in your browser.
3. In the dashboard's **"Host Monitors"** panel, click **"Launch here"** on each
   monitor of this PC you want to fill with the ambient view. Each opens a
   fullscreen window on that monitor and joins automatically.
4. To control from your phone, open the **Dashboard URL** shown in the console
   on your phone's browser (same network), or scan the **QR code** in the
   dashboard's *Join a Screen* panel.

### Option B — from source (developers)
```bash
python -m venv .venv
# Windows:
.venv\Scripts\activate
# macOS/Linux:
source .venv/bin/activate
pip install -r requirements.txt

python run_umbra.py          # full experience (opens dashboard, monitor picker)
# or
python -m conductor.main     # plain headless server, no browser auto-open
```

### Joining another device as a screen
On the other device's browser, open the **Player URL** for the network you share
with the host, e.g. `http://192.168.10.1:8000/join`, then click **Enter**.
(`localhost` only works on the host PC itself — other devices must use the host's
IP.) The console prints a Player URL for **every** network interface; pick the one
on the same network as that device.

## 3. The dashboard

The dashboard is a centered, dark control panel. Top to bottom:

### Emergency bar
- **Blackout All** — fades every screen to black (start of a film).
- **Wake All** — returns every screen to its active visual.

### Brightness & Motion (vertical sliders + quick chips)
- **Brightness** — global light output. Chips: **Off (0%) · Movie (5%) ·
  Ambient (20%) · Bright (60%) · Max (90%)**. Default is **60%** so it's clearly
  visible on a normal monitor; dim to Movie/Ambient for the dark cinematic look.
- **Motion** — drift/animation speed. Chips: **Frozen · Still · Movie ·
  Ambient · Demo**.

> **Why 60% by default?** A browser can't dim your monitor's backlight, so at the
> old "movie" levels the visuals were lost against the screen's lit black. Start
> bright, then dim to taste.

### Sleep states
| State | Behavior |
| :--- | :--- |
| **Awake** | Normal rendering. |
| **Idle Glow** | Theme off; a faint, slow-breathing warm edge glow remains. |
| **Freeze** | Holds the current frame and dims it near black. |
| **Blackout** | Fades to solid black and stops animating. |

### Themes (10, over 6 engines)
| Theme | Engine | Look |
| :--- | :---: | :--- |
| **Cosmic Drift** | E1 | Glowing multi-colored starfield with drifting nebula depth |
| **Dust Motes** | E1 | Soft glowing motes drifting with depth |
| **Abyssal Fluid** | E2 | Swirling domain-warped dark fluid |
| **Volumetric Smoke** | E2 | Vapor drifting along one airflow |
| **Glacial Nebula** | E2 | Slow cold blue-violet clouds |
| **Macro Rain** | E3 | Soft rain streaks down glass |
| **Digital Rainfall** | E3 | Sparse falling glyph columns |
| **The Slow Matrix** | E4 | Dark grid lattice with travelling pulses |
| **Strata Shift** | E5 | Sliding horizontal geological bands |
| **Solar Corona** | E6 | Warm breathing glow hugging the edges |

**Transitions:** switching themes does a smooth **crossfade** (no black gap).
Pick the duration with the **2s / 6s / 10s** chips.

### Presets
One tap sets several parameters at once: **Movie Mode**, **Pre-Movie Ambient**,
**Sleep Glow**, **Blackout**.

### Displays map
A live, Windows-display-settings-style map of the **connected screens**, sized by
resolution and arranged by room layout. **Tap a screen to toggle its animation
on/off** (off = it stays black — e.g. the screen you actually watch movies on).

### Host Monitors
Lists the **physical monitors of the host PC** (auto-detected). **"Launch here"**
opens a fullscreen ambient window on that monitor. "Re-detect" rescans.

### Advanced drawer
- **Cross-screen continuity** ("Treat screens as one room"): when on, world-space
  themes (starfield, fluids, grid, strata) sample a shared room coordinate system
  so the field flows continuously across adjacent screens. Off by default; each
  screen is independent. Edge/column themes (Solar Corona, rain) always stay
  screen-local.
- **Connected Devices**: per-screen **Identify**, **brightness calibration**
  (balance a bright OLED against a dim LCD), **Mark movie** (exclude/blackout),
  and **room layout** inputs (origin X/Y, width/height in room units, rotation
  0/90/180/270°). Saved automatically and restored on reconnect.
- **Join a Screen**: the QR code plus a list of join URLs for every network.

## 4. Networking & security

The server listens on the address in its config (`host`). Two modes:

- **All interfaces (default, `host: "0.0.0.0"`):** reachable on every network the
  PC is on. The console banner and dashboard list a join URL for **each**
  interface — use the one on the same network as the other device.
- **One interface only (recommended for a private link):** restrict the server to
  a single NIC so it isn't exposed on Wi-Fi/the internet at all.

### Locking it to one network (e.g. a direct Ethernet cable)
Create **`data/config.local.json`** (this file is **gitignored**, so your IP never
goes into the repo):
```json
{ "host": "192.168.10.1" }
```
Now the server listens **only** on `192.168.10.1`; other interfaces have no socket
to attack. The banner/dashboard will show just that one URL.

> For a direct cable with no router, give each PC a static IP on the same subnet
> (host `192.168.10.1`, the other PC `192.168.10.2`, mask `255.255.255.0`). From
> the other PC, `ping 192.168.10.1` should reply before you open the browser.

### Firewall — allow only the one device
Windows blocks inbound connections by default. Instead of opening the port to
everything, add a **scoped** rule (elevated PowerShell) that admits only your
peer:
```powershell
New-NetFirewallRule -DisplayName "Umbra (peer only)" -Direction Inbound `
  -Action Allow -Protocol TCP -LocalPort 8000 `
  -LocalAddress 192.168.10.1 -RemoteAddress 192.168.10.2 -Profile Any
```
This permits TCP 8000 **only** from `192.168.10.2` to `192.168.10.1` — nothing
else. Remove it later with
`Remove-NetFirewallRule -DisplayName "Umbra (peer only)"`.

Combined, these give two layers: the server doesn't listen on Wi-Fi, and the
firewall admits a single source IP.

## 5. Under the hood

- **Procedural & light**: visuals are computed live (no stored/looping video).
  RAM stays small and flat; storage use is effectively zero.
- **Clock sync**: the Conductor stamps every state message with its clock; each
  player anchors to it and extrapolates locally, so motion stays aligned across
  screens and keeps advancing between updates. A periodic NTP-style exchange
  measures round-trip time.
- **Performance governor**: if a screen's frame rate drops below ~70% of target
  for ~2s, it lowers internal resolution / particle detail, recovering after ~8s
  of headroom. Throttled to ~12 fps (ambient visuals don't need more).
- **Disconnect safety**: if a screen loses the Conductor for >60s it fades to
  black, then wakes when the connection returns.
- **Debug overlay**: append **`?debug=1`** to a player URL (e.g.
  `http://192.168.10.1:8000/join?debug=1`) to show engines loaded, active theme,
  frame count, fps, and last error — handy if a screen looks wrong. Off by
  default; you can also force it on kiosk windows with `kiosk_debug: true`.

## 6. Configuration files (in `data/`)

| File | Tracked? | Purpose |
| :--- | :--- | :--- |
| `config.json` | yes | Server defaults: port, default theme/brightness/motion, timings, kiosk options. |
| `config.local.json` | **gitignored** | Per-machine overrides (e.g. the bind `host`). Create it yourself. |
| `presets.json` | yes | Named presets. |
| `devices.json` | gitignored | Auto-saved per-device names, calibration, roles, layouts. |

Useful `config.json` keys: `host`, `port`, `default_brightness`,
`default_motion_scale`, `default_theme`, `open_dashboard_on_start`,
`auto_launch_kiosk` (open a window on *every* monitor at startup), `kiosk_debug`.

## 7. Building the Windows executable

From an activated venv in the project root:

```powershell
pip install pyinstaller

python -m PyInstaller run_umbra.py --name umbra --console --noconfirm --clean `
  --add-data "player;player" --add-data "dashboard;dashboard" --add-data "shared;shared" `
  --collect-all uvicorn --collect-all websockets --collect-all h11 `
  --collect-all segno --collect-all screeninfo --hidden-import httptools
```

Then copy the runtime data next to the exe and ship the whole folder:
```powershell
Copy-Item data dist\umbra\data -Recurse
```
(After the first build, rebuild quickly with `python -m PyInstaller umbra.spec --noconfirm`.)

Distribute the entire **`dist\umbra\`** folder (the `.exe` needs `_internal\` and
`data\` beside it). `run_umbra.py` resolves bundled assets and writable data
correctly when frozen, and opens the dashboard on launch.

Notes:
- The exe is unsigned, so SmartScreen may warn ("More info → Run anyway").
- If you locked the bind IP via `config.local.json`, that file travels in
  `dist\umbra\data\` — fine for your machine, and it stays out of the repo.

## 8. Troubleshooting

| Symptom | Fix |
| :--- | :--- |
| Other device can't open `/join` | Use the host's **IP** (not `localhost`); same network; add the scoped firewall rule. `ping` the host first. |
| Dashboard shows "No screens connected" | A screen only appears after a browser opens `/join` (or you click **Launch here**). The dashboard isn't a screen. |
| Screen is black / nothing animates | Raise **Brightness** (try Max); open `…/join?debug=1` and check `theme`, `frames`, `lastError`. |
| Changes don't take effect | Make sure only **one** `umbra.exe` is running (an old one can hold port 8000). The app sends no-cache headers, so a hard refresh isn't usually needed. |
| Multiple network URLs shown | Pick the IP on the same network as the other device (e.g. the Ethernet IP for a direct cable). |
