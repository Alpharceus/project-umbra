# Project Umbra

**A LAN-based ambient display system — a procedural "screensaver" served over your network.**

Umbra turns spare screens (second monitors, laptops, tablets, TVs) into a
synchronized field of dark, slow, low-contrast visuals: environmental rather than
attention-grabbing, with no dependence on matching screen sizes or aspect ratios.

One PC runs the server; every other screen is just a **web browser** pointed at a
URL. Nothing is installed on the other devices, and the visuals are generated by
math in each browser (procedural) — so there's almost no network traffic and no
stored video.

## Architecture

```
              +-------------------------------+
              |  Conductor (Python server)    |
              |  one PC · port 8000           |
              +-------------------------------+
                          | LAN (HTTP + WebSocket)
        +-----------------+------------------+
        v                 v                  v
   Player screen     Player screen       Dashboard
   (browser /join)   (browser /join)     (phone /dashboard)
```

- **Conductor** — FastAPI/uvicorn server. Single source of truth for theme,
  brightness, motion, timing, sleep state, devices. Serves the apps over HTTP and
  syncs over WebSocket.
- **Players** — browser pages (`/join`) that render the active theme locally on a
  Canvas. Opt-in per screen; no install.
- **Dashboard** — a mobile-friendly browser page (`/dashboard`) that controls
  everything.

## Features

- **Screensaver mode** — the app opens the dashboard on launch; pick which of the
  host's monitors to fill from the **Host Monitors** panel (one fullscreen window
  per monitor, auto-join). Multi-monitor supported.
- **10 themes over 6 rendering engines** — glowing starfield, dust motes, advected
  noise fluids/smoke/nebula, rain, glyph rain, grid pulses, sliding strata, warm
  edge corona. Seamless crossfade between themes.
- **Global controls** — brightness, motion, contrast, density; sleep states
  (Awake / Idle Glow / Freeze / Blackout); named presets.
- **Displays map** — a Windows-display-settings-style view of connected screens;
  tap one to toggle its animation on/off (e.g. exclude the movie screen).
- **Per-device** calibration, room layout (position/size/rotation), identify, and
  **cross-screen continuity** so world-space themes flow across adjacent screens.
- **Geometry-agnostic, deterministic, synchronized** rendering with clock sync, a
  local performance governor, throttled ~12 fps loop, and disconnect-safe fade.
- **Network-aware** — binds all interfaces by default, or lock it to one NIC; QR
  join; lists a join URL per interface.

## Quick start

### Run the packaged app (Windows)
Double-click `dist/umbra/umbra.exe`. The dashboard opens automatically; use
**Host Monitors → Launch here** to fill a monitor, and open the printed URL on
other devices to add them.

### Run from source
```bash
python -m venv .venv
.venv\Scripts\activate            # Windows  (source .venv/bin/activate on macOS/Linux)
pip install -r requirements.txt

python run_umbra.py               # full experience (opens dashboard + monitor picker)
# or
python -m conductor.main          # plain headless server
```
Then open the printed URLs:
- Dashboard: `http://<host-ip>:8000/dashboard`
- Player:    `http://<host-ip>:8000/join`  (use the host's IP on other devices, not `localhost`)

See **[USER_MANUAL.md](USER_MANUAL.md)** for full operation, networking/security,
configuration, and the executable build instructions.

## Project structure

```
project umbra/
├── conductor/                  # Python backend (the Conductor)
│   ├── main.py                 # FastAPI app, lifespan, routes, WS endpoints
│   ├── config.py               # settings (config.json + gitignored config.local.json)
│   ├── netutil.py              # interface IP discovery / join URLs
│   ├── kiosk.py                # monitor detection + fullscreen window launch
│   ├── state.py                # UmbraState + StateEngine (versioned, authoritative)
│   ├── devices.py              # device registry (roles, calibration, layout, persistence)
│   ├── sync/                   # websocket.py · messages.py · clock.py
│   ├── routes/                 # api.py · player.py · dashboard.py
│   └── persistence/store.py    # JSON file storage
├── player/                     # browser player (renderer)
│   ├── index.html · css/ · js/
│   └── js/engines/             # engine-base + E1–E6 + noise (themes/registry.js maps them)
├── dashboard/                  # mobile control dashboard (index.html · css/ · js/)
├── shared/js/                  # constants.js · protocol.js
├── data/                       # config.json · presets.json (+ gitignored config.local.json, devices.json)
├── tests/                      # pytest suites (state, sync, devices, integration)
├── run_umbra.py                # entry point (used for the frozen exe; opens dashboard)
├── requirements.txt · pyproject.toml · USER_MANUAL.md
```
